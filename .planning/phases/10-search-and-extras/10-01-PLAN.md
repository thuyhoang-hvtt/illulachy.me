  <action>

   Run `cd apps/blog && pnpm add -D pagefind` from the repo root.

2. Add `postbuild` script to `apps/blog/package.json` scripts section:
   `"postbuild": "pagefind --site dist"`
   This runs automatically after `pnpm run build` completes, generating `dist/pagefind/pagefind.js` and the WASM index bundle.

3. Create `apps/blog/src/components/SearchBar.astro` with custom Stitch-styled UI per D-01 and D-02.

The file has an empty Astro frontmatter fence (just a comment) and a template + script section:

TEMPLATE: A container div with class `search-container bg-surface-container rounded-md px-4 py-3 mb-12` containing:
- An `input#search-input` of type="search" with placeholder="Search posts...", class `w-full bg-transparent text-text-primary placeholder:text-text-tertiary outline-none font-heading text-base`, aria-label="Search posts", autocomplete="off"
- A `div#search-results` with class `mt-4 flex flex-col gap-3 empty:mt-0` and aria-live="polite"

SCRIPT: A self-invoking async `initSearch()` function:
- Dynamic import: `const pagefind = await import('/pagefind/pagefind.js').catch(() => null)` — guards against dev mode where pagefind.js doesn't exist
- If `!pagefind`, return early (graceful dev mode degradation)
- Get `input` element as HTMLInputElement and `results` div
- Listen on `input` event:
  - Get trimmed query; if empty, clear results innerHTML and return
  - Call `const search = await pagefind.debouncedSearch(query)` (Pagefind built-in debounce, NOT raw `search()`)
  - If `!search`, return
  - Load top 8 results: `const loaded = await Promise.all(search.results.slice(0, 8).map((r: any) => r.data()))`
  - Render each result as an anchor element:
    - Outer: `a` tag with href=`${r.url}`, class `block rounded-md px-3 py-2 -mx-1 hover:bg-surface-container-high transition-colors duration-150`
    - Title div: class `font-heading font-semibold text-text-primary`, innerHTML = `r.meta?.title ?? ''`
    - Excerpt div: class `text-sm text-text-secondary mt-1 line-clamp-2`, innerHTML = `r.excerpt` (Pagefind already wraps matched terms in mark tags)

Key implementation notes:
- Do NOT import pagefind-ui.css or pagefind-ui.js (D-02 forbids default UI)
- Use only Stitch design tokens for colors (bg-surface-container, text-text-primary, etc.)
- The mark tags in r.excerpt get default browser highlight styling
