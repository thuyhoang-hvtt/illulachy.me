## Spacing Scale

Declared values (8-point grid, multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`--spacing-1`) | Icon gaps, metadata inline separators |
| sm | 8px (`--spacing-2`) | Inline element spacing, tag gaps |
| md | 16px (`--spacing-4`) | Default element stack spacing (list items) |
| lg | 24px (`--spacing-6`) | Card/row internal padding |
| xl | 32px (`--spacing-8`) | Min screen-edge padding on desktop (D-11) |
| 2xl | 48px (`--spacing-12`) | Section vertical breathing room (`--space-breath`) |
| 3xl | 64px (`--spacing-16`) | Major section breaks between post groups |

Exceptions:
- Prose max-width: 65ch (CONTEXT.md D-11, REQUIREMENTS.md VIS-02) — not a spacing token, a width constraint
- Thumbnail size: 120–160px wide, 80–120px tall (fixed dimension, not from spacing scale)
- Touch targets: 44px minimum (`--touch-target-min`) for any interactive element on mobile

Source: apps/portfolio/src/index.css spacing scale, CONTEXT.md D-11

## Typography

Four roles declared. Two weights max (regular 400 + semibold 600).

| Role | Font | Size | Weight | Line Height | Letter Spacing | Usage |
|------|------|------|--------|-------------|----------------|-------|
| Display | Noto Serif | 48px (`--text-5xl`) | 600 (semibold) | 1.2 (`--leading-tight`) | -0.05em (`--tracking-tighter`) | Page-level blog title only (site name in header) |
| Heading | Noto Serif | 36px (`--text-4xl`) | 600 (semibold) | 1.25 (`--leading-tight`) | -0.025em (`--tracking-tight`) | Post list titles (h2); post page h1; prose h2–h3 inherit from `.prose` plugin defaults |
| Body | Space Grotesk | 16px (`--text-base`) | 400 (regular) | 1.5 (`--leading-normal`) | 0 | Post excerpts on list page; full post prose body |
| Label | Space Grotesk | 14px (`--text-sm`) | 400 (regular) | 1.5 (`--leading-normal`) | 0.025em (`--tracking-wide`) | Dates, reading time, category tags, metadata row |

Declared size scale (4 values): 14px, 16px, 36px, 48px.

Additional: Code blocks use JetBrains Mono at 14px (`--text-sm`), weight 400, line-height 1.625 (`--leading-relaxed`). This reuses the Label size token — no fifth size value is introduced.

Fluid typography: The display heading only uses `clamp()`. All other sizes are fixed px values — Tailwind v4 utility classes applied, not CSS clamp, for body/label/heading in blog context.

Prose headings (h2–h3 inside post body): inherit from `@tailwindcss/typography` `.prose` plugin defaults. No additional declared size token is added; executor must not introduce new px values beyond the 4-size scale above.

Source: apps/portfolio/src/index.css typography section, CONTEXT.md D-08, DESIGN.md §3

## Color

Dark mode is the default. Light mode activates via `@media (prefers-color-scheme: light)` — zero JavaScript (CONTEXT.md D-07).

### Dark Mode (default)

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) — Page surface | `--surface-default` | `#131313` | Page background, prose area background |
| Secondary (30%) — Elevated surfaces | `--surface-container-low` | `#1C1C1C` | Post card hover state, code block background |
| Accent (10%) — Mauve | `--interactive-default` | `#E0AFFF` | Category tag text, hyperlinks in prose, post title hover color |
| Text primary | `--text-primary` | `#FFFFFF` | Post titles, headings, pull-quotes |
| Text secondary | `--text-secondary` | `#A1A1AA` | Post excerpts, body prose |
| Text tertiary | `--text-tertiary` | `#71717A` | Dates, reading time, metadata labels |
| Destructive | `--error` | `#EF4444` | Not used in Phase 8 (no destructive actions) |

### Light Mode (prefers-color-scheme: light override)

| Role | Hex | Usage |
|------|-----|-------|
| Dominant (60%) — Page surface | `#FAFAFA` | Page background |
| Secondary (30%) — Elevated | `#F0F0F0` | Post card hover, code block background |
| Accent (10%) — Mauve | `#9B59D4` (darkened for light contrast) | Category tag text, links (must pass AA on #FAFAFA) |
| Text primary | `#09090B` (`--text-inverse`) | Post titles, headings |
| Text secondary | `#3F3F46` | Excerpts, prose body |
| Text tertiary | `#71717A` | Dates, metadata |

Note: REQUIREMENTS.md VIS-01 references "Catppuccin Mocha" but CONTEXT.md D-07 and .stich/DESIGN.md explicitly supersede this with the custom Stitch editorial palette (#131313 dark / #FAFAFA light). Executor follows Stitch palette.

Accent reserved for: post title hover state, in-prose hyperlinks, category tag text/background pill, focus-visible outlines.
Accent is NOT used for: decorative separators, backgrounds, headings at rest, metadata text.

No-Line Rule (CONTEXT.md D-09, DESIGN.md §2): Zero 1px solid borders for content separation. Use tonal surface shifts (`surface-default` → `surface-container-low`) and negative space only. Ghost border (`rgba(255,255,255,0.06)`) permitted only where accessibility requires a visible container boundary.

Source: apps/portfolio/src/index.css color tokens, CONTEXT.md D-07, D-09, DESIGN.md §2

## Post List Page Layout

Visual reference: `.stich/layout.jpeg` (CONTEXT.md D-14)

Primary visual anchor: post title (Noto Serif Heading role, left column top) — largest element per row, first point of eye contact.

Each PostCard row:
- **Left column (flex-1):** Post title (Heading role, Noto Serif 36px semibold), excerpt (Body role, Space Grotesk 16px, 2 lines max, `line-clamp-2`), metadata row (Label role: date — reading time — category tag pill)
- **Right column (fixed width 120–160px):** Square or 16:9 thumbnail image. When no thumbnail image exists, column collapses (no placeholder box)
- **Row interaction:** On hover, background shifts from `surface-default` (#131313) to `surface-container-low` (#1C1C1C). Transition: 150ms ease-out. No shadow change, no border appears.
- **Row separation:** 32px (`--spacing-8`) vertical gap between rows. No divider lines.
- **Page container:** max-width 680px, centered, horizontal padding 32px desktop / 16px mobile

Category tag pill: `--interactive-bg-subtle` background (`rgba(224,175,255,0.1)`), `--interactive-default` text (`#E0AFFF`), `--radius-full` border-radius, 4px/8px padding (vertical/horizontal), Label typography.

## Post Page Layout

- Prose container: max-width 65ch, centered, horizontal padding 32px desktop / 16px mobile
- Post `<h1>`: Noto Serif 36px (`--text-4xl`) semibold, color `--text-primary`, line-height 1.2, letter-spacing -0.025em
- Reading time: Label role (Space Grotesk 14px, `--text-tertiary`), displayed below the h1, above the prose body. Format: "N min read"
- Body prose: `@tailwindcss/typography` `.prose` class with custom overrides for Stitch tokens. Headings inherit Noto Serif; body inherits Space Grotesk; links use `--interactive-default` mauve.
- Code blocks: Shiki `illu-dark` / `illu-light` custom themes. Background: `--surface-container-low` (#1C1C1C dark / #F0F0F0 light). Padding: 16px (`--spacing-4`). Border-radius: 6px (`--radius-md`). Font: JetBrains Mono 14px.
- Inline code: `--surface-container-low` background, `--text-primary` text, 2px/6px padding, `--radius-default` (4px).

## Accessibility Contract

- All text must meet WCAG AA contrast (4.5:1 for body, 3:1 for large text ≥18px bold)
- Focus-visible ring: 2px solid `--interactive-default` (#E0AFFF), 4px offset (`--outline-focus-offset`)
- Images: `alt` attribute required on all `<img>` elements. Decorative images use `alt=""`
- Semantic HTML: `<article>` per post card, `<time datetime="...">` for dates, `<main>` for page content region
- Touch targets: minimum 44px for any interactive element (`--touch-target-min`)
- Heading hierarchy: Each page has exactly one `<h1>`. Post list uses `<h2>` for post titles. Post page uses `<h1>` for post title, `<h2>`–`<h4>` for prose headings.
