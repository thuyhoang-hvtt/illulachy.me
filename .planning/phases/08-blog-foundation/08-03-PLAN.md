Create `apps/blog/src/pages/[slug].astro` (per RESEARCH.md Pattern 3):

```astro
---
import matter from 'gray-matter'
import { unified } from 'unified'
import remarkParse from 'remark-parse'
import remarkGfm from 'remark-gfm'
import remarkRehype from 'remark-rehype'
import rehypeStringify from 'rehype-stringify'
import rehypeShiki from '@shikijs/rehype'
import illuDark from '../shiki/illu-dark.json'
import illuLight from '../shiki/illu-light.json'
import PostLayout from '../layouts/PostLayout.astro'
import { calcReadingTime } from '../lib/reading-time'

export async function getStaticPaths() {
  const rawFiles = import.meta.glob(
    '../../../../packages/content/content/posts/**/*.md',
    { eager: true, query: '?raw', import: 'default' }
  )
  return Object.entries(rawFiles).map(([path, raw]) => ({
    params: { slug: path.split('/').pop()!.replace('.md', '') },
    props: { raw: raw as string },
  }))
}

const { raw } = Astro.props
const { data: frontmatter, content: body } = matter(raw)
const readingTime = calcReadingTime(body)

const processor = unified()
  .use(remarkParse)
  .use(remarkGfm)
  .use(remarkRehype, { allowDangerousHtml: true })
  .use(rehypeShiki, {
    // D-17: Custom Shiki themes from Stitch palette are the PRIMARY implementation.
    // Import the custom theme JSON files directly -- @shikijs/rehype accepts theme objects.
    themes: {
      light: illuLight,
      dark: illuDark,
    },
  })
  .use(rehypeStringify, { allowDangerousHtml: true })

const html = String(await processor.process(body))
---

<PostLayout frontmatter={frontmatter} readingTime={readingTime}>
  <Fragment set:html={html} />
</PostLayout>
```

CRITICAL IMPLEMENTATION NOTES:

1. **Glob path**: Same 4-level depth as index.astro: `../../../../packages/content/content/posts/**/*.md`. MUST be string literal.

2. **Raw file loading**: Use `{ eager: true, query: '?raw', import: 'default' }` to get raw string content. This is Vite's way to import files as raw text. If this syntax doesn't work in the Astro version, try the alternative: `{ eager: true, as: 'raw' }` (older Vite API). Test both if needed.

3. **gray-matter**: Parses the raw markdown string to extract `data` (frontmatter object) and `content` (body without frontmatter). gray-matter must be explicitly in blog's package.json deps (added in Plan 01).

4. **Unified pipeline**: `remarkParse` -> `remarkGfm` (GitHub Flavored Markdown: tables, task lists, strikethrough) -> `remarkRehype` (markdown AST to HTML AST) -> `rehypeShiki` (syntax highlighting) -> `rehypeStringify` (HTML string output).

5. **Shiki theme in rehype (per D-17 -- custom themes REQUIRED)**: Import the custom theme JSON files and pass them directly to `@shikijs/rehype`:
   ```typescript
   import illuDark from '../shiki/illu-dark.json'
   import illuLight from '../shiki/illu-light.json'
   // ...
   .use(rehypeShiki, { themes: { light: illuLight, dark: illuDark } })
   ```
   This is the PRIMARY implementation path. The custom themes satisfy D-17 (custom Shiki theme matching Stitch tokens).
   - LAST-RESORT FALLBACK ONLY: If `@shikijs/rehype` rejects JSON theme objects at runtime (throws an error), fall back to `github-light`/`github-dark` built-in themes and rely on the CSS variable switching in global.css. Add a comment: `// TODO: D-17 requires custom themes -- fix JSON import format`.
   - The Astro markdown.shikiConfig in astro.config.ts only applies to Astro's built-in markdown pipeline, NOT to this manual unified pipeline. So the rehypeShiki plugin needs its own theme config.

6. **`set:html`**: Astro directive that injects pre-rendered HTML. Use `<Fragment set:html={html} />` to avoid adding an extra wrapper div. The prose styling is handled by PostLayout's wrapper div.

7. **Frontmatter type safety**: `matter(raw).data` returns `Record<string, any>`. PostLayout expects `{ title, date, excerpt, category }`. Cast or validate as needed. For Phase 8, a simple cast is sufficient since we control the content files.

8. **Date normalization**: All post files use ISO 8601 dates (set in Plan 01). `new Date("2024-01-15")` works reliably across environments.

9. **Post not found**: Astro's static generation means only valid slugs get pages. Invalid URLs naturally 404. A custom 404 page is Phase 9 (TAX-04).
