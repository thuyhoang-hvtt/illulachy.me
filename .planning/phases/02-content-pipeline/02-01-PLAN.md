<!-- Pattern from research: Generator structure -->
From research:
- gray-matter for YAML parsing: `matter(content)` returns `{ data, isEmpty }`
- zod for validation: `frontmatterSchema.safeParse(data)`
- fast-glob for discovery: `fg('content/**/*.md')`
- Date normalization: `new Date(dateStr + ' UTC').toISOString()`

<task type="auto" tdd="true">
  <name>Task 2: Create generator script with frontmatter parsing and validation</name>
  <files>scripts/generate-timeline.ts</files>
  <behavior>
    - Parses YAML frontmatter with gray-matter
    - Validates required fields (type, date, title) with zod
    - Normalizes dates to ISO 8601 with UTC
    - Generates IDs from filenames
    - Detects duplicate IDs
    - Filters draft entries
    - Outputs sorted JSON to public/timeline.json
    - Fails fast with clear error on validation failure
  </behavior>
  <action>
    Create `scripts/generate-timeline.ts` implementing the complete generator:
    
    1. **Imports:** gray-matter, fast-glob, zod, fs/promises, path
    
    2. **Zod schema** (per CONTEXT.md):
       ```typescript
       const frontmatterSchema = z.object({
         type: z.string(),           // Free-form, not enum
         title: z.string(),
         date: z.string(),
         url: z.string().url().optional(),
         thumbnail: z.string().url().optional(),
         draft: z.boolean().optional().default(false),
         institution: z.string().optional(),
         tech: z.string().optional(),
         description: z.string().optional(),
       })
       ```
    
    3. **Main flow:**
       - Discover files: `fg('content/**/*.md')`
       - For each file:
         - Read file content
         - Parse with `matter()`, check `isEmpty`
         - Validate with zod schema, `safeParse()`
         - Skip if `draft: true`
         - Generate ID from filename (without .md)
         - Check ID uniqueness (fail on duplicate)
         - Normalize date: `new Date(date + ' UTC').toISOString()`
         - Build ContentNode with all fields
       - Sort by date (chronological, oldest first)
       - Output: `{ nodes, lastUpdated: new Date().toISOString() }`
       - Write to `public/timeline.json` (pretty-printed)
    
    4. **Error handling:**
       - Log `[Timeline] Error in {filepath}: {message}`
       - Exit with code 1 on any error (fail fast)
       - Console colors: Red for errors (if supported)
    
    5. Add npm script: `"generate-timeline": "npx tsx scripts/generate-timeline.ts"`
  </action>
  <verify>
    <automated>mkdir -p content/2024 && echo -e '---\ntype: test\ntitle: Test\ndate: January 2024\n---' > content/2024/test.md && npm run generate-timeline && cat public/timeline.json && rm content/2024/test.md</automated>
  </verify>
  <done>Generator script parses frontmatter, validates fields, normalizes dates, outputs sorted JSON</done>
</task>
